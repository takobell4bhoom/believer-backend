## Standard Response Shape
Success:   { "data": <payload>,  "error": null,      "meta": {} }
Error:     { "data": null,       "error": "message", "meta": {} }
Paginated: { "data": [...],      "error": null,
             "meta": { "page": 1, "limit": 20, "total": 100 } }

## Auth Header
Authorization: Bearer <jwt_token>
All protected routes use: fastify.authenticate hook

## Standard Error Codes
400 validation_error  — request body/params failed validation
401 unauthorized      — token missing, invalid, or expired
403 forbidden         — valid token, insufficient role/permission
404 not_found         — requested resource does not exist
409 conflict          — duplicate resource (e.g. already registered)
500 server_error      — unexpected internal error

## Flutter Client Notes
- Base URL: AppConfig.apiBaseUrl
- All calls via: lib/services/api_client.dart
- Error type: ApiException(message, statusCode)
- Always unwrap: response['data'] — never the root response object
